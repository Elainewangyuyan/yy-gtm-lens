# YY GTM Lens

YY GTM Lens is a Codex skill for researching AI-native companies through a GTM, growth, organization, and career-opportunity lens.

它适合用来做中文 AI 公司深度研究：先基于公开来源完成事实研究，再做研究质检、隐私隔离、网页可视化和 PDF 输出。核心受众是 AI startup founders、GTM / growth / marketing operators，以及想系统判断高潜 AI 公司机会的人。

## What it produces

Given an AI-native company or AI business unit, the skill can generate:

1. `article.md` — a Chinese research article, usually 5,000–8,000 Chinese characters.
2. `website/` — a responsive visual website.
3. `report.pdf` — a public PDF report.

Research QA, visual QA, private section 09, and publish-bundle checks are internal workflow artifacts. They are not shown as public example outputs.

## Latest-date research

This skill is not a live company database. It is a research workflow, QA standard, and visual report generator.

If the runtime has web access, the report should use the execution date as the research cutoff and re-check current public sources such as company websites, product pages, pricing pages, careers pages, repositories, filings, investor materials, and reputable media.

If the runtime has no web access, the output cannot be guaranteed to reflect the latest facts.

Dynamic facts should include access dates. Unverifiable facts should be marked as unknown instead of guessed.

## Install

Copy the skill folder into your Codex skills directory:

```bash
cp -R yy-gtm-lens ~/.codex/skills/yy-gtm-lens
```

Then restart Codex if the skill does not appear immediately.

## Use

Example prompt:

```text
Use $yy-gtm-lens to research Glean and generate the Chinese article, website, and public PDF.
```

For a text-first workflow:

```text
Use $yy-gtm-lens to research Perplexity. Stop after the reviewed public/private Markdown and QA memo; do not render the website yet.
```

## Example websites

On GitHub, clicking `website/index.html` in the file browser shows source code. To let users open the examples as real websites, enable GitHub Pages:

1. Go to `Settings` → `Pages`.
2. Under `Build and deployment`, choose `Deploy from a branch`.
3. Branch: `main`.
4. Folder: `/ (root)`.
5. Save.

## Privacy rule

Section 09 is private. It should never be included in:

- public websites;
- public PDFs;
- GitHub publish bundles;
- hidden DOM nodes;
- JavaScript config;
- public source maps.

CSS hiding or unlinked URLs do not count as privacy protection. The private section must be physically separated.

## Disclaimer

The reports generated by this skill are research notes, not investment, legal, financial, or employment advice. Always verify high-risk claims from primary sources before making decisions.
