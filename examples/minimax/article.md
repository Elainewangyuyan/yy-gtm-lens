# MiniMax 深度研究

- 研究截止：2026-06-25
- 核心受众：AI startup founders、GTM 负责人及高潜 AI 公司求职者

## 01 总览与前世今生

**判断：MiniMax 已经不是一家靠 Talkie 或海螺视频讲故事的消费应用公司，而是同时经营“模型、应用、开发者平台”三层业务的上市 AI 公司。它最突出的优势是全球化收入和全模态产品矩阵；最核心的压力是高研发投入、持续亏损，以及产品线太宽带来的品牌和资源聚焦问题。**

MiniMax 于 2022 年初成立，创始人闫俊杰此前在商汤工作超过六年，曾任副总裁及研究院副院长。公司很早选择“模型与应用同时做”：先用 Glow、星野和海外 Talkie 验证 AI 陪伴，再用海螺切入视频生成，同时建设语音、音乐和开放平台。从 2026 年官网主推 M3、MiniMax Code、Mavis/Agent Team 与 Token Plan 的变化看，本文判断其战略重心正在向代码、Agent 和生产力迁移，定位从“多模态内容产品公司”进一步走向“AI 时代的平台公司”。[来源 1、3、4]

这条路径与多数基础模型创业公司不同。很多公司先训练通用模型，再寻找应用；MiniMax 从早期就拥有消费产品和海外分发，因而更早得到真实用户反馈、付费数据和内容安全问题。公司 2025 年业绩公告显示，全年收入从 2024 年的 3,052.3 万美元增至 7,903.8 万美元，同比增长 158.9%；73% 收入来自中国内地以外市场。AI 原生产品贡献 67.2% 收入，开放平台及其他企业服务贡献 32.8%。这说明 MiniMax 的商业化不只是未来故事，而是已有两条收入线。[来源 3]

但它仍是高投入的模型公司。2025 年研发费用为 2.5277 亿美元，约为当年收入的 3.2 倍；调整后净亏损为 2.5086 亿美元，与 2024 年基本持平。公司已证明收入可以快速增长，尚未证明收入增长足以覆盖前沿模型、视频和音频等多条研发线。[来源 3]

## 02 赛道与竞争

**判断：竞争强度“极高”。MiniMax 已形成跨模态产品和两类收入来源，但公开数据尚不足以证明它在任何单一战场取得稳定领先。跨产品销售和数据反馈目前应视为待验证的协同假设，而非已经形成的护城河。**

第一场是通用大模型、AI Coding 与 Agent。M3、M2.7 面对 Claude、GPT、Gemini，以及 DeepSeek、Qwen、Kimi、GLM 等模型；MiniMax Code 和 Agent 还要面对 Claude Code、Codex、Cursor、Windsurf、Manus 等产品。这个市场比的是真实软件工程、工具调用、长任务稳定性、上下文成本和开发者生态，而不仅是 benchmark。

第二场是 AI 视频。Hailuo 2.3 面对 Google Veo、OpenAI Sora、Runway、快手可灵、字节 Seedance、Pika 和 Luma。MiniMax 同时提供 API 和消费产品；但视频质量窗口变化快，同行可以迅速追平，版权与角色生成风险也更高。由于各家分辨率、时长、音频、排队优先级与套餐条件不同，本报告不作简单价格排名。

第三场是语音和音乐。Speech 2.8 面对 ElevenLabs、Cartesia、OpenAI、Azure 等语音服务，Music 2.6 面对 Suno、Udio 和传统音乐工具。MiniMax 的优势是语音、视频、音乐可在同一平台采购，适合内容生产和 Agent 工作流；劣势是每个垂直赛道都有专注度更高的强对手。

第四场是 AI 陪伴。Talkie/星野面对 Character.AI、Replika、豆包智能体以及大量角色聊天应用。陪伴产品具有高互动频次和内容供给飞轮，但用户忠诚可能绑定角色而不是公司品牌，内容审核、未成年人保护和 IP 风险会持续增加。

第五场是企业和开发者平台。MiniMax 要争夺的不只是 API 调用，还包括开发者使用其开放权重模型后的官方付费转化、团队订阅和多模态生产工作流。它的相对优势是价格、模态覆盖和中国内地以外收入基础；结构性短板是云生态、企业销售网络和第三方工具入口弱于阿里、腾讯、字节、微软和 Google。

竞争位置可以概括为：MiniMax 是中国少数已经同时证明全球消费收入、企业/API 收入和前沿模型迭代能力的公司之一；但“全模态”既是差异化，也可能变成资源分散。真正的护城河不是模型列表长，而是不同模态是否能共同服务一个高频、可复购的工作流。

## 03 产品与商业模式

**判断：商业模式清晰度“较高”。MiniMax 同时向消费者卖订阅与内容生成额度，向开发者卖模型调用和 Token Plan，向企业卖 API、资源包、稳定性与定制服务。问题不在“怎么赚钱”，而在什么业务最终能形成最大收入和最好毛利。**

产品大致分为四层：

1. 消费应用层：Talkie/星野提供 AI 角色和陪伴；Hailuo AI 提供视频生成；MiniMax Audio 面向语音、克隆和音乐创作。
2. 生产力与 Agent 层：MiniMax Agent/Mavis 和 MiniMax Code 面向代码、办公、研究、内容生产及长时间任务。
3. 开放平台层：提供文本、视频、语音、音乐、图像和 MCP 等 API，支持按量付费、资源包和企业定制。
4. 开放生态层：M2.5 权重、Mini-Agent、CLI、MCP、OpenRoom、skills 等项目承担开发者分发和生态验证。[来源 1、6、13]

2025 年收入结构已经可以验证：AI 原生产品收入为 5,307.5 万美元，同比增长 143.4%；开放平台及企业服务收入为 2,596.3 万美元，同比增长 197.8%。公司解释，前者增长来自用户参与和付费意愿提升，尤其是 Hailuo 的持续商业化；后者来自付费客户增加。公开数据没有披露 Talkie、Hailuo、Audio、Agent 分别贡献多少收入，因此不能把某款热门产品直接视为绝对主力。[来源 3]

截至 2026 年 6 月 25 日，官网价格体系覆盖多种付费习惯。文本模型 API 中，M3 在不超过 512K 输入时，页面标准价为每百万输入 token 0.30 美元、输出 1.20 美元；超过 512K 后为输入 1.20 美元、输出 4.80 美元。M2.7 为 0.30/1.20 美元。Token Plan 面向个人和小团队，Plus、Max、Ultra 页面月价分别为 20、50、120 美元，实际额度和公平使用条件应以购买页为准。视频 API 按时长和分辨率收费，例如 Hailuo 2.3 Fast 的 768P、6 秒视频为 0.19 美元；音乐 2.6 每首最长 5 分钟为 0.15 美元。价格可能随促销和产品迭代变化，不应被视为长期承诺。[来源 8、9]

这套定价有两个信号。第一，MiniMax 在用较低试用门槛和固定月费吸引开发者，可能意在争夺 Claude Code 等工具中的模型替换机会。第二，多模态 API 有机会通过同一客户提高钱包份额，例如一家内容团队可以同时采购文案、配音、音乐和视频。但产品和计费体系过多，也会增加理解成本；如果用户不能快速知道“我应该从哪个产品开始”，丰富会变成转化阻力。

## 04 用户、收入与增长信号

**判断：增长信号“强”，经营透明度“中等”。上市后收入、毛利、地域和业务结构已经公开，但当前 MAU、付费用户、留存、CAC、产品级毛利和客户集中度仍未披露。**

截至 2025 年末，公司披露累计服务超过 2.36 亿个人用户，覆盖 200 多个国家和地区；企业客户及开发者累计达到 21.4 万，覆盖 100 多个国家和地区。需要注意，这是累计服务口径，不等于月活、年度活跃或付费客户。[来源 3]

更有价值的是收入质量变化。2025 年中国内地以外收入占比从 69.8% 提升至 73.0%，说明全球化不只是下载量叙事，而是真实收入来源。毛利率从 12.2% 提升至 25.4%，公司归因于模型与系统效率提升以及基础设施分配优化。销售与分销费用从 8,699.5 万美元降至 5,189.6 万美元，同比下降 40.3%；公司称 AI 原生产品更多由自然增长和用户推荐驱动，推广费用因此下降。收入增长而销售及分销费用下降，是值得关注的经营杠杆信号；但由于公司没有披露 CAC、渠道投入及费用确认节奏，不能直接推导获客效率或单位经济已经改善。[来源 3]

开发者生态也在加速。2026 年 6 月 25 日抓取时，MiniMax GitHub 组织约有 6,700 名关注者；`skills` 仓库约 1.28 万 stars，Mini-Agent 约 2,800，CLI 约 2,000，MiniMax-MCP 约 1,500。M2 在 2025 年末成为首个在 OpenRouter 单日消耗超过 500 亿 token 的中国模型；2026 年 2 月，M2 系列平均日 token 消耗已超过 2025 年 12 月的六倍，Coding Plan token 消耗超过十倍。不过，后两项是公司披露，仍应与官方 API 收入、活跃开发者和续费结合观察。[来源 3、13]

当前最重要的缺失数据是：

- Talkie、Hailuo、Agent、Code 的 MAU、付费率和留存；
- 消费产品与 API 的获客成本和推理毛利；
- 21.4 万企业及开发者中有多少为付费、持续调用和生产环境客户；
- 收入是否集中于少数海外应用或大客户；
- Token Plan 的试用、激活、升级与续费漏斗。

## 05 资本与估值

**判断：资金实力“强”，经营杠杆正在改善，但公司仍处于高亏损阶段。当前估值高低需要结合 2026 年 6 月 25 日附近市值、完全摊薄股本和未来收入预期重新计算，本轮暂不作确定评级。**

公司于 2026 年 1 月 9 日在港交所上市，发行价为每股 165 港元；首次发行净募集约 45.961 亿港元，超额配售另募集约 6.973 亿港元，合计净募集约 52.93 亿港元。[来源 3] 本轮已确认的官方年度业绩材料没有完整列出上市后的主要股东、机构持股比例和创始人控制权，因此不沿用百科或二手汇总中的投资人名单。进入网页/PDF 前，应以招股书的 Pre-IPO Investments、Substantial Shareholders 及加权投票权章节补齐资本结构。

截至 2025 年末，公司现金及现金等价物、理财资产、受限现金和定期存款合计约 10.503 亿美元，高于 2024 年末的 8.806 亿美元。由于 IPO 发生在 2026 年 1 月，上市募集资金属于年末之后事项。上述 10.503 亿美元是流动性资源合计，不等同于全部可自由动用现金。公司仍处于高投入状态：2025 年调整后净亏损约 2.509 亿美元，研发费用约 2.528 亿美元。[来源 3]

净亏损 18.716 亿美元看起来非常夸张，其中 15.899 亿美元来自优先股等金融负债的公允价值重估，并不等于当年现金消耗；观察经营应重点看调整后亏损。调整后亏损只比 2024 年增加 2.7%，而收入增加 158.9%，说明经营杠杆在改善；毛利增长 437%，研发费用增长 33.8%，也支持这一判断。但公司距离自我造血仍远，2025 年收入仅覆盖约三成研发费用。

媒体在 2026 年 3 月业绩发布后曾报道其市值超过 300 亿美元，但这不能直接代表 6 月 25 日估值。由于本轮没有完成同日股价、完全摊薄股本和预期收入倍数的统一计算，报告暂不下“估值偏高/偏低”的结论。更值得观察的是未来四个季度收入增速、毛利率、调整后亏损率和经营现金流。[媒体披露 21]

## 06 模型、数据、评测与开源

**判断：MiniMax 是明确的自研基础模型公司，不是模型封装商。其技术路线从 Lightning Attention、MoE 和超长上下文，发展到面向 Agent 的训练数据、强化学习系统与稀疏注意力；开放权重是开发者获客策略，但不同模型的开放程度需要逐一确认。**

MiniMax-01 技术报告披露，Text-01 采用 32 专家的 MoE，总参数 4,560 亿、每 token 激活约 459 亿，并将 Lightning Attention 与 MoE 结合；训练上下文达到 100 万 token，推理可外推至 400 万。M2 系列进一步转向“真实 Agent 工作流”，旗舰 M2 总参数约 2,299 亿、激活约 98 亿，通过可执行环境中的代码和办公任务轨迹训练，并建设 Forge 强化学习系统。M2.7 甚至让模型参与训练实验监控、日志分析、代码修改和评测循环。[来源 14、15]

2026 年 6 月发布的 M3 引入 MiniMax Sparse Attention，API 支持最高 100 万上下文，并从训练早期进行原生多模态联合训练。官网展示 BrowseComp、软件工程、终端任务、长时间论文复现和 CUDA 优化等结果，但这些主要是公司自测或公司选择的 benchmark，需要第三方跨模型复现。截至 2026 年 6 月 25 日，MiniMax 已在 Hugging Face 提供 M3 权重、配置与推理文件，许可证标记为 MiniMax Community License。M3 可以称为“权重已公开/open-weight”，但不宜直接等同于 OSI 定义的开源软件；商业使用条件仍需按许可证核验。[来源 10、17]

MiniMax 拥有四类潜在产品反馈与评测场景：

1. Talkie/星野提供长对话、角色一致性和情感互动信号；
2. Hailuo、Audio 和 Music 提供创作偏好、生成反馈和多模态任务数据；
3. API、Token Plan、CLI 与开发工具提供真实代码、工具调用和长任务轨迹；
4. 公司内部 Agent 已覆盖接近 90% 员工，应用于研发、数据分析、运营、招聘、销售与市场，为组织工作流提供内部实验场。[来源 3]

这些产品能够产生使用反馈，但公开资料不足以确认相关数据是否、以及在何种授权条件下用于模型训练。公司也没有完整披露训练数据清单、授权比例、用户数据进入训练的规则和去重机制。考虑到版权诉讼及外部提出的模型蒸馏指控，数据治理、来源证明和训练合规会成为上市公司级别的持续风险。

## 07 团队、组织与招聘

**判断：组织结构比多数未上市 AI 创业公司透明。核心决策高度集中在创始人闫俊杰和 COO 云晔漪；技术负责人进入董事及高管体系，产品与商业化有明确最高负责人，但 COO 以下的增长、营销和商业化汇报线仍未公开。**

闫俊杰现任创始人、董事长、CEO 和 CTO，同时负责整体管理、业务运营、财务、AI 研发战略及重大决策。高度集中的优点是模型、产品和资本可以快速统一；风险是公司产品线扩张后，决策负荷和关键人依赖很高。[来源 4]

云晔漪任执行董事和 COO，负责整体管理与业务运营，并特别负责产品和商业化战略。她曾在商汤从事融资、战略投资、CEO 助理、战略和创新业务。这意味着如果寻找营销、增长、运营或 GTM 负责人，公开信息中最可能的最高汇报节点是 COO，而不是单独披露的 CMO/CRO。赵鹏宇负责大语言模型研究与工程，周彧聪负责视觉模型研究与工程。[来源 4]

截至 2025 年末，公司有 428 名全职员工，全年员工薪酬及股份支付约 8,430 万美元。公司招聘官网同时开放顶尖人才计划、2026 校招、2027 实习和社会招聘，并强调技术、产品、内容、审美之间的跨界影响。官网没有公开可抓取的完整岗位清单，但可以确认公司仍在持续招聘；动态岗位应以飞书招聘页为准。[来源 3、5]

从业务结构判断，MiniMax 至少需要五类 GTM 能力：

- 全球消费产品增长：Talkie、Hailuo、Audio 的订阅、内容生态和留存；
- 开发者增长：M3/M2、CLI、MCP、开源仓库到 Token Plan/API 的转化；
- 企业商业化：API、大客户资源包、解决方案和客户成功；
- 创作者生态：视频、语音、音乐模板、案例和版权治理；
- 投资者与公司传播：上市后的业绩、风险和全球品牌叙事。

真正需要通过访谈确认的是：这些团队是否统一归 COO，产品负责人能否影响定价和分发，增长岗位是否拥有跨产品数据，以及国际市场是按产品还是按区域组织。

## 08 GTM 难题与低成本增长建议

**判断：MiniMax 的核心 GTM 问题不是没有产品，而是产品太多、公司品牌被产品品牌切散。近期应先用低成本内容和开发者实验验证“开源/工具入口 → 首次高价值任务 → Token Plan/API 付费”，再决定是否投入统一账号、跨产品数据和完整行业工作流。**

### 难题一：公司品牌与产品品牌割裂

Talkie、Hailuo、MiniMax Audio、Agent、Code 都有独立心智，但普通用户未必知道它们来自同一家公司。品牌割裂有利于针对不同人群，却削弱交叉销售。

建议（中高成本）：先做统一任务导航原型，再评估是否建设统一账号。 不先要求用户理解全部产品，而是从“我要做视频”“我要写代码”“我要做播客”“我要搭建 Agent”进入，再引导到对应产品。只有原型能明显提高跨产品激活率和付费转化，才进入账号、权限和数据打通。

### 难题二：模型声量未必进入官方付费漏斗

开源权重、GitHub skills、CLI、MCP 和第三方 Coding 工具能快速带来开发者注意，但用户可能自部署或通过聚合平台调用，不一定给 MiniMax 贡献收入。

建议（低成本）：把“可复现任务包”作为增长单位。 围绕代码迁移、长文档分析、自动生成测试、视频批量本地化等任务，提供公开仓库、输入样例、成本计算器和一键 Token Plan 配置。不要只宣传 benchmark，要展示“用多少时间、多少 token、多少钱完成什么工作”。指标看样例仓库到 API Key 创建、首次成功任务和四周复用的转化。

### 难题三：全模态优势没有被包装成业务结果

同时拥有文本、视频、语音和音乐很稀缺，但官网仍以模型列表呈现，客户容易逐项比价。

建议（先低成本验证、后高成本产品化）：先用服务化方式测试一个端到端行业场景。 可从电商广告本地化、游戏角色内容生产、短剧/知识视频生产中选择一个，用模板、人工编排和现有 API 验证质量、频次和付费；只有复购成立，再开发完整工作流产品。

### 难题四：中国内地以外收入高，合规风险也更复杂

73% 收入来自中国内地以外市场，是优势，也是版权、儿童安全、角色内容和地缘政治风险的放大器。2025 年 6 月，Disney 与 Universal 联合就 Hailuo 相关服务对 MiniMax 提起版权侵权诉讼；Warner Bros. Discovery 随后另行提起诉讼，相关主张仍需等待司法程序确认。2026 年 2 月，Business Insider 报道称 Anthropic 指控 MiniMax 等公司对 Claude 进行模型蒸馏。本轮未检索到可核验的 MiniMax 公开回应。上述均属于外部指控，不代表法院、监管机构或独立调查已经认定事实。[媒体披露 18、19、20]

建议（中高成本）：先公开规则与响应流程，再逐步产品化合规能力。 低成本起点是透明度页面、IP 投诉入口、企业使用指南和事件响应 SLA；后续再建设角色/IP 黑名单、生成记录、品牌安全模式、地区策略和审计导出。合规不只是法务成本，也可以成为视频和陪伴产品的企业卖点。

### 难题五：上市后容易被短期指标牵着走

公司需要同时维持模型迭代、收入增长、毛利改善和公开市场预期。如果每条产品线都追求增长，组织会重新变成资源争夺。

建议（中成本）：先统一最小事件口径，再设任务级经营看板。 每个产品不只报告用户和收入，还报告成功任务数、单位任务推理成本、四周复用、跨模态采用和付费回收期。用共同指标决定资源，而不是比较下载量和发布声量。

### 建议执行顺序

建议由 2–3 人跨职能小组负责：一名 Developer Growth/产品营销 owner、一名数据或增长产品伙伴、一名开发者工程师。

1. 0–30 天： 选一个代码任务包，补齐样例仓库、成本计算器和邮件/社区触达。核心指标是首次成功任务率；若样例访问到成功任务低于预设基线，则停止扩量、先修产品体验。
2. 31–60 天： 选择一个全模态行业场景，用现有 API 和人工服务交付 5–10 个设计伙伴。核心指标是二次使用和付费意愿；无复用则不开发重产品。
3. 61–90 天： 统一 GitHub、CLI、MCP、Token Plan 和 API 的最小事件口径，验证首次成功任务、四周复用及付费升级。
4. 验证后再扩大： 只有留存和单位经济成立，才投入统一账号、模板市场、完整工作流与行业伙伴计划。

## 10 催化剂、风险与持续观察

**判断：未来 12 个月最值得观察的是 M3 能否把模型声量转成 Token Plan/API 收入，以及全模态产品能否继续提高毛利。公司值得持续跟踪，但高市值已经提前计入了很强的增长预期。**

上行催化剂包括：

- M3 公开权重能否扩大第三方部署和开发者采用，并进一步转化为 Token Plan/API 收入；
- MiniMax Code/Mavis 在真实代码和办公任务中形成高留存；
- Token Plan 将开源和第三方工具用户转化为直接付费；
- Hailuo、Speech、Music 组合成可复购的企业内容工作流；
- 毛利率继续提升，销售费用保持克制，调整后亏损率明显下降；
- 上市带来的品牌和资本能力帮助其获取全球大客户。

核心风险包括：

- 前沿模型、视频、语音和音乐多线研发导致现金消耗长期高企；
- OpenAI、Anthropic、Google 与中国大厂发动价格战；
- 开源模型带来使用量，却无法回流官方收入；
- Talkie 与内容生成产品面临版权、角色安全和未成年人风险；
- Anthropic 蒸馏指控与地缘政治限制可能影响国际平台和企业客户；
- 创始人高度集中决策，产品线扩张后组织复杂度上升；
- 若市场估值持续快于收入基本面，公开市场压力可能影响长期研发选择。

建议持续跟踪七个指标：季度收入增速、中国内地以外收入占比、毛利率、调整后亏损率、Token Plan/API 收入、核心产品四周留存、企业付费客户数。MiniMax 已经证明它能把 AI 能力做成收入，但下一阶段需要证明的是：能否从多个成功产品，进化为一个具有复利的统一平台。
